# dgis-ml-models

YOLO-based computer vision models for the DGIS autonomous drone system.

Two models are trained and maintained here:
- **Sim model** — trained on synthetic images generated inside the Unity simulation, used for in-sim species detection
- **Mobile model** — trained on real-world images, deployed on-device on the Android smartphone mounted to the quadruped robot

## Setup

```bash
git clone https://github.com/your-org/dgis-ml-models
cd dgis-ml-models

# Create and activate virtual environment
python -m venv yolo_env

# Windows
yolo_env\Scripts\activate

# macOS / Linux
source yolo_env/bin/activate

# Install dependencies
pip install -r requirements.txt

# PyTorch with CUDA 12.1 (if using GPU)
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

---

## Training

```bash
# Train the simulation model
python scripts/train_unity.py

# Train the mobile model
python scripts/train_real.py
```

Training outputs dynamically land in `runs/detect/train/`.

---

## Exporting

The training scripts (`train_unity.py` and `train_real.py`) are configured to automatically export the model to the appropriate format (`.onnx` or `.tflite`) at the end of the training run.

The newly exported weights will be saved in `runs/detect/train/weights/`. You should then copy them to the `models/` directory for consumption by other DGIS components.

## Dataset Notes

Images are tracked via **Git LFS** (see `.gitattributes`). Labels (`.txt` files) are committed directly since they're small.

- `Dataset_Unity/` — synthetic images generated by the DGIS Unity simulation's YOLO data generation pipeline
- `Dataset_Real/` — The model was trained on a custom , unified dataset that we created by gathering and merging multiple data sources. You can view, explore, and download the dataset here:

* **[Link to the Dataset on Roboflow](https://app.roboflow.com/omar-hafez-agi0o/venom-v5/1))**
- `data/` — the active merged split used during a training run (can be a symlink or copy of either dataset)
